# モデル比較ベンチマーク (メモ)

narashi の表記ゆれ統合で使う埋め込みモデルを、同じ「物差し」で横並び比較した記録です。
モデルを差し替えるたびにこの手順 (`.claude/skills/model-eval`) を再実行し、本表を更新してください。

## 物差し

- **評価データ**: `tests/data/glossary.txt`(日本語の表記ゆれ + 日英中のクロス言語同義 + 難しい単独語)
  - 同一グループ内ペア = 統合すべき(正例)、異グループ間ペア = 統合すべきでない(負例)
- **実行**: `cargo run --example benchmark -- <model> [threshold]`(指標ロジックは `src/eval.rs`)
- **既定閾値**: `DEFAULT_THRESHOLD = 70.0`(CLI のデフォルト値)
- **モデル間比較は最適閾値で行う**: 固定の 70.0 での横並び比較は校正(`cos_baseline`)依存で各モデルの最適動作点を反映しないため意味が薄い。本記録では best-thr / clusterF1 真ピーク(=各モデルの最適閾値での値)で比較する
- 指標の意味:
  - **best-F1 / best-thr**: 分類 F1 を最大化する閾値とその値(各モデルを最適閾値で測った素ペア性能の上限)
  - **margin**: `正例min − 負例max`。0 に近いほど正例・負例の分離が良い(完全分離なら正)
  - **clusterF1(真ピーク)**: `normalize`(union-find の推移閉包込み)の実出力と正解グループのペア F1 = **実運用の挙動**。各モデルの最適閾値(1 刻みスイープのピーク)で比較する
  - **誤統合(false merges)**: `normalize` の推移閉包込みの実出力で、**異グループの語を同一クラスタにしてしまったペア数**(= データ破壊)。表記ゆれ統合では誤統合は取りこぼしより痛いため、clusterF1 が同程度なら誤統合が少ないモデルを選ぶ。`benchmark` は既定閾値での件数と具体例、各スイープ行の件数を出力する
  - **per_text**: 1 用語あたり埋め込み時間(load 時間はサンドボックスのコールドスタート依存のため speed は per_text で比較)

### 1 刻みスイープでの真のピーク(モデル間比較の主表)

各モデルは最適閾値が異なるため、固定閾値での比較は不公平になる。`cargo run --example fine_sweep -- <model>`
で **1 刻み(40〜95)** に測り直し、各モデルの本当のピーク(clusterF1)とその最適閾値で比較する。これが
実運用挙動でのモデル間比較の主表。

「誤統合(@70)」は既定閾値 70 での `normalize` 実出力(推移閉包込み)の誤統合ペア数。clusterF1 と並ぶ
採用判断の主要指標(— は未測定)。

| モデル | **1 刻みピーク** | 最適閾値 | ピーク時 P / R | 誤統合(@70) |
|---|---:|---:|---:|---:|
| granite-278m-multilingual | **0.705** | 70 | 0.739 / 0.673 | 24 |
| **gte-multilingual-base(既定)** | 0.682 | 70 | **0.782** / 0.604 | **17** |
| **distiluse-multilingual-v2** | 0.667 | 70 | 0.945 / 0.515 | — |
| granite-311m-multilingual-r2 | 0.663 | 88 | 0.824 / 0.554 | — |
| LaBSE【見送り】 | 0.662 | 89 | 1.000 / 0.495 | — |
| granite-107m-multilingual | 0.652 | 74 | 0.738 / 0.584 | — |
| e5-large | 0.644 | 72 | 1.000 / 0.475 | — |
| paraphrase-MiniLM-L12 | 0.626 | 89 | 0.718 / 0.554 | — |
| e5-small | 0.616 | 71 | 1.000 / 0.446 | — |
| granite-97m-multilingual-r2 | 0.551 | 79 | 0.782 / 0.426 | — |
| snowflake-arctic-embed-m-v2.0【非推奨】 | 0.512 | 57 | — | — |

> e5-base・mpnet は 1 刻み未測定。5 刻みスイープ表でのピークはそれぞれ 0.508(@65)・0.588(@85)で、
> いずれも上位群に届かない。

→ **重要**: 1 刻みで測り直すと各モデルが最適閾値で評価される。clusterF1 真ピークだけ見ると
**granite-278m が最高(0.705)**で gte(0.682)を約 0.023 上回り、distiluse(0.667)・e5-large(0.644)が続く。
なお各モデルの最適閾値は校正(`cos_baseline`)で動かせるため、最適点が 70 か否かは優劣ではない。

→ **ただし採用は誤統合(データ破壊)の少なさを重視して gte を既定に据える**。閾値 70 での誤統合は
gte 17 件に対し granite-278m は 24 件、素ペア分類の誤統合(FP)では gte 2 件に対し granite 19 件(約 10 倍)。
granite は「猫=犬」のようなクロス言語・近縁語の誤統合が増える(高い再現率の裏返し)。表記ゆれ統合では
誤統合は取りこぼしより痛いため、clusterF1 で僅かに劣っても誤統合の少ない gte を既定とし、granite は
clusterF1/再現率を最優先する場合の選択肢(`--model granite`)とする。

→ **ピーク帯の形**: granite-278m は 70 が頂点(0.705)で 65〜78 の広い帯で 0.62〜0.705 を維持。gte も鋭い
ピーク(70 で 0.682)を持ち 64〜78 で 0.63〜0.68 と安定。E5 系は閾値 70 付近に鋭いピークを持ち、超えると
適合率は上がる(75 で P=1.0)が再現率が急落する。paraphrase 系はピークが高閾値側(89)にある。

## 評価できなかった候補(配布形式の制約)

ONNX(≤2GB・外部重み `model.onnx_data` なし)** しか読み込めない。既定 gte を超える/軽量枠でより良い
多言語モデルがないか調査した。IBM Granite Embedding 系(Apache 2.0・単一ファイル ONNX)を評価し、
**granite-278m は clusterF1 で gte を上回ったが誤統合が多く、既定は gte のまま据え置き**選択肢として追加した。
有力候補の一部は配布形式の制約で評価できなかった。

## 所見

- **既定は gte-multilingual-base(誤統合の少なさ最優先)**。clusterF1 真ピーク 0.682(granite に次ぐ 2 位)を、
  全モデル中で最も高い適合率(ピーク時 P=0.782、素ペア分類では誤統合 2 件・P=0.964、閾値 70 の
  クラスタ誤統合 17 件)で達成する。純粋な分類性能 best-F1=0.776 も最良。表記ゆれ統合では「異なる語を
  誤って統合する(=データ破壊)」が取りこぼしより痛いため、誤統合の少なさを最優先して既定に据える。
  コストは 1.2GB・推論やや低速。fastembed の `try_new_from_user_defined` で読み込む(`--model gte`、
  `cos_baseline=0.42` 校正で最適閾値は CLI 既定 70)。
- **clusterF1/再現率を最優先するなら granite-278m-multilingual(選択肢)**。clusterF1 真ピーク **0.705** と
  全モデル中で最高で、gte(0.682)に約 0.023 上回り、ピーク時 P/R=0.739/0.673 と再現率を gte より大きく
  改善(0.604→0.673)。完全一致グループも 17/26 と最多。IBM Granite Embedding R1(XLM-RoBERTa 系・
  Apache 2.0)で 200+ 言語のうち日本語は明示的な学習対象。約 1.1GB で gte(1.2GB)より小さく、推論も
  同一セッション比較で gte より速い(≈3.9ms vs ≈4.7ms)。**ただし閾値 70 での誤統合が 24 件(gte 17 件)・
  素ペア FP は 19 件(gte 2 件の約 10 倍)**で「猫=犬」型のデータ破壊が増えるため、既定にはせず選択肢に留めた
  (`--model granite`、`cos_baseline=0.44` 校正で最適閾値は CLI 既定 70)。
- **軽量代替の第一候補は distiluse-multilingual-v2**。clusterF1 真ピーク **0.667** を高適合率
  (ピーク時 P/R=0.945/0.515)で達成。gte(0.682)に約 0.015 譲るだけで、e5-large(0.644)・
  e5-small(0.616)を上回る。**最小サイズ(0.54GB)・最速級(2.45ms)**のため、gte が重い/遅い用途での
  軽量な代替として最有力。`--model distiluse`(`cos_baseline=0.39` 校正で最適閾値は 70)。
- **e5-small もなお有力**(最軽量 0.45GB・最速 1.7ms・384次元)。最適閾値 71 でピーク時 P=1.000 と
  最も保守的。clusterF1 真ピーク 0.616 は distiluse(0.667)に約 0.05 譲るが、サイズ・速度を最優先する
  用途や、誤統合を極限まで避けたい場合に好バランス。表記ゆれ統合では「異なる語を誤って統合する
  (=データ破壊)」方が取りこぼしより痛いため、保守性重視ならこちら。
- **閾値を調整できるなら paraphrase-MiniLM @89 も候補**(1 刻みピーク 0.626、最速 1.28ms)。ただし
  baseline 0.30 のままでは最適点が 89 と高く、運用閾値の再設定が前提。gte には僅差で及ばない。
- **e5-large は E5 系の上限で gte に肉薄**。1 刻みピーク 0.644(@72、P=1.0 R=0.475)は gte(0.682)に
  次ぐ 2 位で、5 刻み表の 0.587 は過小評価だった。best-F1(0.646)・分離 margin(−40.7)も E5 系で最良。
  ただしコストは **約 8 倍の推論時間**(13.6ms)+ 2.2GB と最重量で、精度・速度・サイズの総合では gte
  (0.682 / 5.4ms / 1.2GB)に劣後する。
- **e5-base は劣後**(全閾値で e5-small 以下、かつ約 3 倍遅い)。採用理由なし。
- **granite 系の 278m 以外は採用せず**。granite-311m-r2(R2 最新・1.25GB)は 0.663 で適合率は高い
  (P=0.824)がピークが高閾値側(88)で gte 未満、granite-107m(0.43GB)は 0.652、granite-97m-r2
  (0.39GB)は 0.551 と、いずれも 278m・gte に届かない。コードには `UserModel` として残し eval から
  選べる(`--example benchmark -- granite-107m` 等)が CLI 既定の選択肢には出さない。
- **snowflake-arctic-embed-m-v2.0 は【非推奨】**。gte 派生だが検索特化で、本データの精度が低い
  (clusterF1 真ピーク 0.512、gte 0.682 に届かず)。詳細は上の結果表の注記参照。
- **別系統(BGE-zh / 英語 MiniLM / CLIP)は本データでは不適**。多言語(特に日本語)対応が必須条件。
- **配布形式で評価できなかった候補**(snowflake-l / e5-large-instruct / jina-v3 / bge-m3 / ruri-base)は
  上の「評価できなかった候補(配布形式の制約)」を参照。単一ファイル ONNX が用意できれば再評価の余地あり。
- **校正の含意**: E5 系(`cos_baseline=0.70`)は最適閾値が 64–75 と既定 70 付近、paraphrase 系
  (`cos_baseline=0.30`)は 85–90、gte(`cos_baseline=0.42`)は校正済みで 70。paraphrase 系を既定運用するなら
  baseline 引き上げで最適閾値を寄せる再校正が要る。

## 用途別の選び方

| 重視点 | 推奨 | 閾値 |
|---|---|---|
| 誤統合の少なさ最優先(**既定**・高適合率・1.2GB) | **gte-multilingual-base** | 70 |
| clusterF1/再現率最優先(誤統合は増える・1.1GB) | **granite-278m-multilingual** | 70 |
| 精度に近い × 軽量・最速(0.54GB) | **distiluse-multilingual-v2** | 70 |
| 最軽量・最保守(高適合率・0.45GB) | e5-small | 70 |
| 精度ピーク × 最速(閾値調整可) | paraphrase-MiniLM-L12 | ~89 |
| 再現率最優先 | paraphrase-mpnet-base | ~87 |
