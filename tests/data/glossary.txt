# narashi 評価用 用語集 (テストデータ)
#
# モデルを差し替えるたびに同じ物差しで精度を比較するための「正解データ」です。
# 1 行 = 1 つの同義語グループ。メンバはカンマ (,) 区切り。
#   - 同じグループ内のペア  => 統合されるべき (正例)
#   - 異なるグループ間のペア => 統合されるべきでない (負例)
# メンバが 1 つだけの行は「単独語 (ディストラクタ)」で、他のどれとも統合されない。
# `#` 始まりの行と空行は無視されます。用語は自由に追記してください。
#
# narashi は多言語埋め込みを使うため、実運用では日本語・中国語・英語が
# 混在しがちです。各グループは表記ゆれ (かな/カナ/漢字/送り仮名) に加えて、
# 同義の外来語・英語・中国語も同じグループにまとめ、言語をまたぐ統合能力も測ります。
#
# ── ベンチマーク設計方針 (飽和対策) ───────────────────────────────
# 上位モデルが容易に満点(誤統合0・高再現率)に達してしまうと、モデル間の優劣を
# 分離できない(= ベンチマークの飽和)。これを避けるため、本用語集は「易しい例」に
# 加えて、最上位モデルでも誤りうる**難所**を意図的に多く含む:
#   (A) 難正例 = 表層がほとんど重ならない同義語(略語/外来語/漢語/英語/中国語が
#       入り混じり、文字の重なりに頼れない)。再現率(取りこぼしの少なさ)を厳しく問う。
#   (B) 難負例 = 意味的に近接するが統合してはいけない語。埋め込み空間で近くに来るため
#       適合率(誤統合の少なさ)を厳しく問う。次の 3 種を用意:
#         - 共下位語(同カテゴリの兄弟語): 季節・色・曜日・方角・兄弟・飲料 …
#         - 紛らわしい異義語(字面が近い別語): 科学/化学, 保証/保障 …
#         - 対義語(同じ話題で極性だけ反転し空間上で近い): 増加/減少, 拡大/縮小 …
# モデル比較は閾値非依存の「最難ペア(最高スコア負例/最低スコア正例)」と
# clusterF1 真ピークの両方で行う(`src/eval.rs`)。詳細は `docs/benchmarks.md`。
